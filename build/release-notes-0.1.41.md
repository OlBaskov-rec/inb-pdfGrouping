# PDF Grouping 0.1.41

## English

### Changed
- Button renamed: "Create output groups for each range".
- The window now always sizes its height to the content automatically: it is as compact as
  possible while showing everything, so there is no need to drag the bottom edge and nothing can
  be hidden by shrinking it. The height no longer jumps around.

### Fixed
- When a message appears (the red overlap/conflict zone), the page-ranges list still shows a full
  5 rows, all sections keep their drawn borders, and there is no scrollbar in the upper part —
  the message area itself scrolls internally if the text is long, and the bottom "Process" bar
  stays visible.
- The group-creation buttons in the "Group label" column no longer jump around; they keep a
  stable position.

## Русский

### Изменено
- Кнопка переименована: «Создать группы для вывода каждого диапазона».
- Окно теперь всегда подгоняет высоту под содержимое автоматически: оно максимально компактно,
  но показывает всё — тянуть нижний край не нужно, и сжатием ничего не спрячешь. Высота больше
  не «прыгает».

### Исправлено
- При появлении сообщения (красная зона пересечений/конфликта) список диапазонов по-прежнему
  показывает все 5 строк, у всех секций видны прорисованные границы, и нет полосы прокрутки в
  верхней части — длинное сообщение прокручивается внутри своей зоны, а нижний бар «Обработать»
  остаётся виден.
- Кнопки создания групп в колонке «Метка группы» больше не «прыгают» — у них стабильная позиция.
