# PDF Grouping — Разделение и группировка PDF

Программа для Windows, позволяющая разделить PDF-документ на группы страниц по заданным диапазонам и склеить каждую группу в отдельный PDF-файл.

## Возможности

- Поддержка **всех подвидов PDF** (включая защищённые, сжатые, линейные) — используется **qpdf** для надёжного разбора
- Разделение PDF на группы через указание **числовых диапазонов страниц**
- Объединение нескольких диапазонов в один выходной файл с заданной **меткой (буквой/номером)**
- Drag & Drop PDF-файла в окно программы
- Мгновенный просмотр количества страниц
- Понятный графический интерфейс (WPF / .NET 10)

## Требования

- **Windows 10/11** (x64)
- **.NET 10 Desktop Runtime** ([скачать](https://dotnet.microsoft.com/en-us/download/dotnet/10.0))
- **qpdf** — внешняя утилита для надёжного разбора всех подвидов PDF

## Установка и настройка

### 1. Установка .NET 10 Desktop Runtime

Скачайте и установите .NET 10 Desktop Runtime с [официального сайта Microsoft](https://dotnet.microsoft.com/en-us/download/dotnet/10.0). Выберите **x64** версию для Windows.

### 2. Установка qpdf

qpdf — это проверенная open-source утилита для работы с PDF, которая корректно обрабатывает **все подвиды PDF** (линейные, зашифрованные, сжатые объектные потоки и т.д.).

#### Способ A: Скачать вручную

1. Скачайте qpdf с официального сайта: https://github.com/qpdf/qpdf/releases/latest
   - Выберите файл `qpdf-<версия>-bin-mingw64.zip` (64-бит)
2. Распакуйте архив
3. Создайте папку `qpdf` рядом с программой (`PdfGrouping.App.exe`)
4. Скопируйте `qpdf.exe` из распакованного архива (`bin/qpdf.exe`) в папку `qpdf/`

**Структура должна быть:**
```
PdfGrouping.App.exe
qpdf/
  qpdf.exe
```

#### Способ B: Установить глобально

Установите qpdf через winget:
```
winget install qpdf
```
Затем укажите полный путь к `qpdf.exe` в конфигурации или пропишите его в `PATH`.

### 3. Запуск программы

Запустите `PdfGrouping.App.exe`.

## Использование

### Основной рабочий процесс

1. **Откройте PDF-файл** (кнопка «Обзор» или перетащите файл в окно)
2. **Добавьте диапазоны страниц:**
   - Введите начальную и конечную страницу (например, `1–5`)
   - Нажмите «+ Добавить диапазон»
   - Повторите для всех нужных диапазонов
   - Несвязанные диапазоны (например, `1–3` и `10–15`) можно добавить последовательно
3. **Задайте метку группы** (букву или номер: А, Б, 1, Часть1 и т.д.)
4. **Нажмите «Создать группу»**
   - Все текущие диапазоны объединятся в одну группу
   - Диапазоны очистятся — можно вводить следующую группу
5. **Повторите шаги 2–4** для всех нужных групп
6. **Выберите папку для сохранения** результатов
7. **Нажмите «▶ Обработать»**

### Пример

Исходный PDF из 50 страниц нужно разделить на 3 документа:

| Метка | Диапазоны страниц | Результат |
|-------|-------------------|-----------|
| **A** | 1–10, 25–30 | `A.pdf` (16 стр.) |
| **B** | 11–24 | `B.pdf` (14 стр.) |
| **C** | 31–50 | `C.pdf` (20 стр.) |

## Публикация / деплой на внутренний сервер

### Single-file executable

Собрать программу в один автономный .exe файл:
```bash
dotnet publish src/PdfGrouping.App/PdfGrouping.App.csproj -c Release -r win-x64 --self-contained true -p:PublishSingleFile=true -o ./publish
```

После этого папку `publish/` можно:
- Разместить на общем сетевом диске (файловый сервер)
- Упаковать в ZIP и распространять
- Разместить на внутреннем веб-сервере для скачивания

**Важно:** qpdf.exe не входит в single-file exe — его нужно распространять отдельно (см. раздел «Установка qpdf»).

### Публикация через ClickOnce

Для ClickOnce-публикации через Visual Studio:
1. Открыть решение в Visual Studio
2. Project → Publish → ClickOnce
3. Указать URL/FTP на внутреннем сервере
4. Включить автоматические обновления

ClickOnce позволяет пользователям всегда иметь актуальную версию.

## Технические детали

### Архитектура

```
src/
  PdfGrouping.App/           # WPF приложение (.NET 10)
    Models/                   # PageRange, PdfGroup
    Services/                 # PdfService (iText7 + qpdf)
    ViewModels/               # MainViewModel (MVVM)
    Converters/               # WPF value converters
    MainWindow.xaml/.cs       # Главное окно
```

### Используемые технологии и библиотеки

| Компонент | Технология | Назначение |
|-----------|-----------|------------|
| UI | WPF (.NET 10) | Кроссплатформенный UI-фреймворк |
| MVVM | CommunityToolkit.Mvvm 8.4 | Генерация свойств и команд |
| Разбор PDF | **qpdf** (внешний процесс) | Надёжное разделение всех подвидов PDF |
| Чтение/запись PDF | iText7 8.0 | Чтение количества страниц, склейка страниц |

### Почему qpdf, а не iText7 для разделения?

- **iText7** (AGPL-лицензия) отлично справляется с большинством PDF, но может иметь проблемы с некоторыми подвидами (линейные PDF с XRef streams, зашифрованные объектные потоки определенных типов)
- **qpdf** — отраслевой стандарт для структурного разбора PDF, гарантированно обрабатывает **все подвиды**, включая сложные случаи сжатия объектных потоков и нестандартные шифрования
- Гибридный подход: qpdf разделяет, iText7 склеивает — используем лучший инструмент для каждой задачи

## Лицензия

Проект использует:
- **iText7** под лицензией AGPLv3 (требуется открытый исходный код производных работ)
- **qpdf** под лицензией Apache 2.0 (свободное использование)
- **CommunityToolkit.Mvvm** под лицензией MIT
